# AGENTS-local.md (Local / uv mode)

This file is auto-generated by `init-local.sh` and is git-ignored. It tells AI
coding agents how to actually run commands in the project as it is currently
set up. The canonical architectural guidance lives in `AGENTS.md` / `CLAUDE.md`.

This project is running in **local (no-Docker) mode** — Python via `uv`,
SQLite, no Redis, Celery in always-eager mode. Wrap every Python command in
`uv run`. Frontend tooling (`npm`) runs directly on the host.

## Run the dev server

```bash
uv run bash dev.sh
```

`dev.sh` is generated by `init-local.sh` with a randomized port baked in. The
URL is `http://127.0.0.1:<port>` (see the `dev.sh` contents or the init script
output for the actual port).

```bash

uv add --dev <package>        # add a dev dep
```

## Database

Local mode uses SQLite at `db.sqlite3` (no DATABASE_URL is set in `.env`). To
reset, just delete `db.sqlite3` and re-run `uv run python manage.py migrate`.

## Celery

Celery is configured in always-eager mode (`CELERY_TASK_ALWAYS_EAGER=True` in
`.env`). Tasks run synchronously in the calling process — no Redis, no worker.
Email sent via `post_office` will still flow through the eager task and print
to the console (`EMAIL_URL=consolemail://`).
