# AGENTS-local.md (Docker mode)

This file is auto-generated by `init-docker.sh` and is git-ignored. It tells AI
coding agents how to actually run commands in the project as it is currently
set up. The canonical architectural guidance lives in `AGENTS.md` / `CLAUDE.md`.

This project is running in **Docker Compose mode**. Wrap every Django, Tailwind,
and shell command in `docker compose ...`.

## Run the dev server

```bash
docker compose up -d web
```

The randomized port is baked into `dev.sh` (which the `web` service runs as its
`command:`) and the published `127.0.0.1:<port>:<port>` mapping in
`docker-compose.yml`.

## Django management commands

```bash
docker compose run --rm web python manage.py <command>
docker compose run --rm web python manage.py migrate
docker compose run --rm web python manage.py makemigrations
docker compose run --rm web python manage.py shell
```

## Tailwind

```bash
docker compose run --rm web npm run tailwind:build   # one-shot
docker compose run --rm web npm run tailwind:watch   # watch mode
docker compose run --rm web python manage.py generate_tailwind_directories
```

## Tests

```bash
docker compose run --rm web python manage.py test
```

## Open a shell

```bash
docker compose run --rm web bash
```

## Bring everything up / down

```bash
docker compose up -d
docker compose down
docker compose logs -f web
```

## Celery

Celery worker and beat run as separate services (`celery`, `celery-beat`) and
are started by `docker compose up -d`. Use Redis (the `redis` service) as the
broker — no eager mode in this setup.
